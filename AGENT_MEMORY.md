## ============================================================
## 项目全貌
## ============================================================

### 论文标题
"Context-Conditional Feature Learning for Task-Agnostic Continual Memory Retrieval"

### 投稿目标
IJCNN (International Joint Conference on Neural Networks), CCF-C类会议
录用率约39%，录用阈值约3.0/5分

### 核心创新点（3个）

1. **CCFL编码器**: f(x,c) = g(h(x) || e(c))
   - 同一输入+不同上下文 → 不同特征表示
   - 用监督对比损失 + 组合标签(c*K+y)训练
   - 神经科学动机：MEC→DG通路（上下文信号→模式分离）

2. **Multi-Hypothesis Decoding (MHD)**
   - 测试时遍历所有已知上下文，选择最匹配的
   - 实现task-agnostic推理，无需test-time上下文ID
   - O(C)复杂度，C=20时仍100%推理准确率

3. **Dual-Head Architecture + Triple Regularization**
   - 分类头cls(h(x))：标准持续学习（CA3模式完成）
   - 记忆头f(h(x),e(c))：上下文条件化检索（CA1上下文检索）
   - 三重防护：EWC(参数正则) + Replay(数据正则) + CCFL(结构正则)

### 1. Context-Dependent MNIST（核心实验）
场景：相同数字(0-4)在多个上下文中有不同标签

| 上下文数 | CCFL-T | CCFL-I(MHD) | 上下文推理 |
|---------|--------|-------------|-----------|
| 5       | 99.0%  | 99.0%       | 99.8%     |
| 10      | 99.3%  | 99.3%       | 100.0%    |
| 20      | 99.1%  | 99.1%       | 100.0%    |

无上下文基线: ≤20%（随机水平）

### 2. Split-MNIST（5个任务，每任务2类）

| 方法          | 平均准确率 | 平均遗忘 |
|--------------|-----------|---------|
| Fine-Tune    | 19.7%     | 79.7%   |
| EWC          | 22.3%     | 77.6%   |
| LwF          | 35.2%     | 64.8%   |
| SI           | 19.9%     | 80.1%   |
| MAS          | 34.0%     | 66.0%   |
| ER           | 85.1%     | 14.9%   |
| CCFL+EWC-Cls | 76.9%     | 23.1%   |
| CCFL+EWC-Mem | 97.4%     | 0.2%    |

### 3. Permuted-MNIST（10个任务，随机置换）

| 方法          | 平均准确率 |
|--------------|-----------|
| EWC          | 95.4%     |
| CCFL+EWC-Cls | 92.8%     |
| CCFL+EWC-Mem | 94.0%     |

注意：CCFL比EWC低1.4%，这是因为Permuted-MNIST每个任务输入完全不同，
没有上下文冲突，EWC的参数保护就够了。CCFL的优势在于共享输入+上下文依赖标签。

### 4. 消融实验（Split-MNIST）

| 变体                    | 准确率 | 变化    |
|------------------------|-------|---------|
| 完整模型                 | 98.8% | ---     |
| 去掉EWC                | 97.0% | -1.8%   |
| 去掉Replay             | 96.4% | -2.4%   |
| 去掉对比损失(CTR)        | 54.8% | -44.0%  |
| 去掉CCFL(EWC+Replay only)| 79.6% | -19.2%  |

关键发现：对比损失是最关键组件（-44%），CCFL条件化贡献+19.2%

### 5. 遗忘曲线
- CCFL+EWC+Replay: 平均遗忘0.2%
- Fine-Tune: 平均遗忘79.7%

### 6. Replay Buffer敏感性
5样本/类: 98.2% → 20样本/类: 99.2%（饱和）

### 7. CIFAR-100（CPU上跑的，结果不够好，需要GPU重跑）
- ER: 2.7%（太低，说明训练setup有问题）
- CCFL-Cls: 4.0%
- CCFL-T: 27.2%（比ER好10倍，但绝对值太低）

### 8. 理论验证（5个定理）
- T1: 上下文分离界 — gap=0.74, σ=0.03, 界紧
- T2: 样本复杂度 — 5样本/上下文即可
- T3: 检索精度界 — 类间分离1.41
- T4: 噪声退化 — 线性退化
- T5: 上下文容量 — 20个上下文仍100%推理

## ============================================================
## 关键架构细节
## ============================================================

### CCFL模型结构
```
输入x → backbone h(x) → 拼接 [h(x), e(c)] → fusion g() → 上下文条件化特征z
                                      ↓
                              context_embedding e(c)

分类头: cls(h(x)) → 100类（CIFAR-100）
记忆头: f(h(x), e(c)) → 128维特征 → kNN检索
```

### 训练损失
L = 0.5 * L_ctr + L_cls + 0.5 * L_ctr_replay + L_cls_replay + L_EWC

其中：
- L_ctr: 监督对比损失，标签为 c*100+y（CIFAR-100）或 c*10+y（MNIST）
- L_cls: 交叉熵分类损失
- L_EWC: EWC正则化项，λ=5000

### 关键超参数
- 对比损失温度 τ = 0.07
- EWC λ = 5000
- Replay buffer: 20 samples/class (MNIST), 5-10 samples/class (CIFAR-100)
- Adam优化器, lr=0.001
- 训练5 epochs per task (MNIST), 5-10 epochs (CIFAR-100)

## 审稿人可能攻击的点 & 应对策略
## ============================================================

### 1. "CIFAR-100结果太弱"
应对：用GPU+ResNet-18重跑，目标50%+。强调相对优势（CCFL >> ER）而非绝对值。

### 2. "缺少DER++/SCR等强基线"
应对：如果时间允许，添加DER++基线。否则在论文中说明"we focus on representative
methods from each category (regularization, replay, distillation)"。

### 3. "理论证明不严格"
应对：在论文中提供完整证明（至少T1-T3）。当前只有proof sketch。

### 4. "Context-Dependent MNIST不是标准benchmark"
应对：强调这是本文提出的新评估场景，对应真实世界的上下文依赖问题（机器人、
自动驾驶等）。标准benchmark（Split/Permuted-MNIST）也做了。

### 5. "MHD就是穷举搜索"
应对：O(C)复杂度在实践中可接受（C≤20）。未来可扩展为层次化上下文聚类。

### 6. "与CoCoOp太相似"
应对：Table 2已列出6个维度的差异。核心区别：特征空间 vs prompt空间，
CL vs 单任务，有理论保证 vs 无。
